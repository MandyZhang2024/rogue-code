2) 四大核心业务板块

明确“交易能力 + 审核要求 + 交付逻辑”，避免走向通用电商模板。

A. 核心交易

技能交易板块

  • 服务类型：单次服务 / 包时服务 / 定制项目
  • 技能范围：微调、Agent 定制、爬虫、固件、LLM 应用、前后端、提示词工程、AR 程序等
  • 流程：需求澄清 → 里程碑交付 → 验收 → 售后申诉
  • 定价：固定价 / 阶梯套餐 / 报价单
  • 交付：平台文件 + 仓库交付记录；技术维度评分
B. 核心交易

AI 软件 / Agent 交易板块

  • 资产类型：私有 Agent、脚本、本地 AI 工具、权重、RAG 系统、工作流等
  • 强制授权类型：开源免费 / 单次授权 / 商用永久授权 / 源码买断
  • 功能：版本管理、更新日志、License 展示、兼容环境声明、在线预览
  • 交付：部署包 / 源码包 / API 接入说明 / 模型依赖说明
C. 免费开源

开源免费代码区

  • 零交易零抽成，仅做共享与协作
  • 支持：代码、固件、Agent 工程、硬件图纸
  • 强制字段：MIT/GPL/Apache 协议、运行说明、禁止付费捆绑声明
  • 功能：Fork 镜像、版本记录、贡献者榜、Issue 讨论、二创展示墙
D. 核心交易

DIY AI 硬件交易板块

  • 仅允许个人 DIY/手工改造硬件，工业量产禁售
  • 品类：ESP32 设备、AR 眼镜、离线 AI 终端、嵌入式视觉设备、DIY 机器人等
  • 上架必备：焊接图纸、固件说明、实拍图、制作教程

3) 双交易体系完整规则

模式 规则 结算与平台收益 交付机制
现金第三方支付交易 接入支付宝 / 微信支付 / Stripe；下单担保、自动确认、售后申诉。 平台抽固定服务费；资金走第三方担保，平台不直触用户资金。 数字资产 T+3 自动确认；技能服务 T+5；硬件签收后 T+3。
极客以物易物（无现金） 支持跨品类置换：硬件↔Agent、技能↔硬件、代码↔设备。 无资金流水，无抽成，仅留仲裁记录与信用评分。 实物邮寄（物流单+签收）或数字直传（哈希+下载记录）。
置换流程：提案匹配 → 双向确认 → 平台托管交付凭证 → 完成交付 → 双方互评。

6) 支付接入、资金流转、提现逻辑

支付与结算链路

  • 统一支付网关：AliPay / WeChat / Stripe Adapter
  • 资金链路：买家付款至第三方担保 → 达成条件后结算 → 平台抽佣 → 卖家入账
  • 提现余额仅来自已结算订单净额
  • 风控冻结覆盖争议单、异常退款、高风险账户

提现与对账规则

  • 提现周期：T+1 / T+3 可配置
  • 提现失败回退：原路退回平台余额并消息通知
  • 支付回调自动对账 + 每日结算报表
  • 异常补单任务由异步队列重试并告警

7) 风控、版权、硬件合规、AI 代码安全

版权与授权治理

  • 强制授权来源与 License 声明
  • AI 模型/数据集来源与使用范围披露
  • 侵权投诉流程：通知 → 举证 → 裁定 → 下架/恢复

代码与硬件安全

  • 上架包自动扫描：木马、后门、恶意下载器、敏感权限调用
  • 高危资产必须提交最小权限运行建议
  • 硬件强制 DIY 自制声明，禁止工业量产倒卖
  • 电池/射频/激光等高风险项强提示与免责确认
交易风控重点：刷单、洗稿、重复上架、欺诈置换；新账号限额，高风险延迟放款并触发人工复核。

9) 后端基础技术架构选型建议（开源友好）

层级 建议 说明
架构模式模块化单体起步先控复杂度，再按交易/社区/审核服务化拆分。
后端框架NestJS 或 FastAPI开源生态成熟，便于快速迭代和 API 组织。
数据与缓存PostgreSQL + Redis交易主库 + 会话/队列/缓存。
搜索OpenSearch / Elasticsearch标签、全文、语义检索统一入口。
对象存储S3 兼容（MinIO 起步）资产包、固件、图纸、截图统一管理。
异步任务BullMQ / Celery审核扫描、支付对账、消息通知。
安全扫描ClamAV + 规则引擎对代码压缩包和上传内容做自动安全检查。
支付网关Provider Adapter + Webhook 总线支持多支付通道统一接入与回调处理。
权限RBAC + 资产级 ACL身份、类目、风险等级联合控制。
可观测性Prometheus + Grafana + OTel交易链路与审核链路可追踪。
部署Docker + Kubernetes（可选）+ Terraform开源优先，兼顾云上扩展。
协作接口GitHub/GitLab OAuth仓库元数据同步，增强开源协作体验。
